案例1:指定yum软件源
为 server0 指定可用的 yum 软件源
C YUM软件库的地址为 http://classroom.example.com/content/rhel7.0/x86_64/dvd/
C 将此配置为虚拟机 server0 的默认软件仓库
C 确认可用的仓库列表
C 利用yum仓库安装xeyes
vim /etc/yum.repos.d/dvd.repo写入以下内容：
[dvd]
name=dvd
baseurl=http://classroom.example.com/content/rhel7.0/x86_64/dvd/
enabled=1
gpgcheck=0
命令行执行：
yum clean all
yum repolist
yum -y install xeyes


案例2：chmod权限设置
    1）以root用户新建/nsddir/目录，在该目录下新建文件readme.txt
mkdir /nsddir
touch /nsddir/readme.txt
    2）使用户zhangsan能够在/nsddir/目录下创建/删除子目录
chmod o+wx /nsddir   
 3）使用户zhangsan能够修改/nsddir/readme.txt文件的容
  chmod o+w /nsddir/readme.txt
 
案例3：chown归属设置
    1）新建/tarena1目录
mkdir /tarena1
	a）将属主设为gelin01，属组设为tarena组
	chown gelin01:tarena /tarena1
b）使用户gelin01对此目录具有rwx权限，其他人对此目录无任何权限
chmod 750 /tarena1
    2）使用户gelin02能进入、查看/tarena1文件夹（提示：将gelin02加入所属组）
gpasswd -a gelin02 tarena1
    3）新建/tarena2目录
mkdir /tarena2
	a）将属组设为tarena
chown :tarena /tarena2	
b）使tarena组的任何用户都能在此目录下创建、删除文件
chmod g+rwx /tarena2
    4）新建/tarena/public目录
mkdir /tarena/public
	a）使任何用户对此目录都有rwx权限
chmod 777 /tarena/public	
b）拒绝zhangsan进入此目录，对此目录无任何权限
setfacl -m u:zhangsan:--- /tarena/public

案例4:权限设置
	1、创建文件夹/data/test,设置目录的访问权限，使所有者和所属组具备读写执行的权限；其他人无任何权限。
mkdir -p /data/test
chmod 770 /data/test
	2、递归修改文件夹/data/test的归属使所有者为zhangsan，所属组为tarena。
chown -R zhangsan:tarena /data/test
	3、请实现在test目录下，新建的所有子文件或目录的所属组都会是tarena。
chmod g+s /data/test
	4、为lisi创建ACL访问权限，使得lisi可以查看/etc/shadow文件
setfacl -m u:lisi:r /etc/shadow
案例5：绑定到LDAP验证服务

 C  使用系统 classroom.example.com 提供的LDAP服务
 C  验证服务的基本DN是：dc=example,dc=com
 C  账户信息和验证信息都是由 LDAP 提供的
 C  连接要使用证书加密，证书可以在下面的链接下载：
      http://classroom.example.com/pub/example-ca.crt 
 C  确认本地可以识别ldapuser0用户
命令行运行：
yum -y install sssd
authconfig-tui
进入文本编辑：
使用空格键选择：
使用LDAP
使用LDAP认证
进入下一步：
选择使用TLS
服务器：classroom.example.com
基础DN：dc=example,dc=com
选择确认，进入下一个界面，注意千万不要点击确定，然后再次打开一个终端，在命令行输入：
cd /etc/openldap/cacerts
wget http://classroom.example.com/pub/example-ca.crt
下载成功之后，切换回第一个终端，点击确定，然后在命令行执行：
systemctl start sssd
systemctl enable sssd
id ldapuser0
